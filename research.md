# 모네뮤직 카운터 관리시스템 — 리서치 정리

---

## 핵심 철학: 결제 = 스케줄 등록

수강생이 결제하는 순간, 4주치 수업 날짜가 자동으로 생성된다.
이 날짜 리스트가 시스템 전체의 원천 데이터이며, 모든 기능은 여기서 파생된다.
**"결제일 기준 스케줄로 제어할 수 있느냐"** 가 시스템 포함 여부의 기준이다.

---

## 원천 데이터 (결제 시 확정되는 것들)

- 수강생 이름 (상담원이 직접 입력, 동명이인은 상담원이 구분하여 입력)
- 담당 선생님
- 수업 요일 / 시간
- 과목 (클린보컬 / 언클린보컬 / 기타 / 작곡 / 믹싱)
- 수업 횟수 (1개월=4회, 3개월=12회, 믹싱=4개월=16회)
- 수강 기간 시작일 ~ 종료일
- 적용 할인 내역
- 결제 수단 (카드 / 이체 / 현금)
- 결제 유형 (신규 / 연장 / 예약금 / 마이너스수업)

---

## 수강생 DB 항목

수강생 1인당 기록되어야 할 정보:

- 이름
- 나이
- 전화번호
- 등록 상태 (재원 / 휴원 / 퇴원 / 등록대기 / 미등록)
- 과목
- 담당 선생님
- 정규 수업 요일 / 시간
- 최초 등록일
- 찾아온 이유 (상담 시 수집)
- 본인의 문제점 (상담 시 수집)
- 하고 싶은 것 (상담 시 수집)
- 랭크 (1차전직 / 2차전직) → 2차 전직서 수령 시 자동 변경
- 수강 동의서 수령 여부 (O/X)
- 2차 전직서 수령 여부 (O/X) → 수령 시 랭크 자동 변경
- 커버 촬영 여부 (O/X)
- 직전 휴/퇴원일
- 예측 복귀 여부
- 예측 휴원 사유 (본심)
- 수강생이 언급한 사유
- 연락 기한
- 환불 휴원 여부
- 남은 수강 횟수
- 지인 할인 연결 정보 (누구의 추천인지)
- 후기 할인 적용 여부 + 후기 URL + 작성 기한
- 인터뷰 할인 적용 여부 + 촬영 완료 여부
- 상품권 발급 여부 / 발급일 / 사용 여부 / 사용 클래스 / 만료일
- 마이너스 수업 기록 (미결제 상태로 차감된 횟수, 결제 완료 시 자동 해소)
- 자차 보유 여부 (O/X) → 시간대별 주차 필요 대수 자동 계산에 사용
- 결제 기준 재원 기간 (결제 이력 기반 자동 누적, 4주=1개월 기준. 예: 4개월 2주)
- 출결 코드 (전화번호 뒷 4자리 기본)
- 메모 (특이사항)

## 출결 코드 규칙

수강생이 출석 시 전화번호 기반 코드를 키패드에 입력하는 방식.

**코드 부여 규칙**
- 기본: 전화번호 뒷 4자리
- 재원 중인 수강생 간 중복 불가 (휴원 중인 수강생과는 겹쳐도 무방, 출결패드에 미노출)
- 중복 발생 시 조정 순서:
  1. 뒷 4자리 → 12344 (뒷자리 한 번 더 반복해서 5자리)
  2. 그것도 겹치면 → 중간 자리 활용
- 코드 조정은 상담원이 수동 입력
- 시스템은 재원 수강생 간 실시간 중복 여부를 체크하여 상담원에게 알림

**휴원자 복귀 시**
- 복귀 시점에 기존 코드가 현재 재원 수강생과 겹치는지 자동 체크
- 겹치면 상담원에게 알림 → 상담원이 새 코드 수동 입력

---

## 출석 키패드 시스템

별도의 키패드 화면이 존재하며, 수강생이 직접 코드를 입력하여 출석 처리.

- 수강생이 출결 코드(전화번호 기반)를 키패드에 입력
- 입력 즉시 해당 수강생 출석 처리 및 횟수 차감
- 재원 수강생 간 코드 유일성이 보장되므로 중복 출석 없음

---

## 시간대별 주차 대수 자동 계산

- 수강생 DB의 자차 보유 여부(O/X) 기반
- 해당 시간대 출석 예정인 수강생 중 자차 보유가 O인 인원 수를 자동 집계
- 시간표에서 시간대별 주차 필요 대수가 자동 표시
- 상담원이 주차 현황을 실시간으로 파악하여 조정 가능

---

## 알림 구조

- 개근 달성, 보강/패스 처리, 상품권 지급 등 모든 이벤트 알림은 **담당 선생님에게 직접 가지 않음**
- 시스템은 상담원에게만 알림
- 상담원이 확인 후 담당 선생님에게 직접 전달하는 구조

## 파생 모듈 1 — 수업 날짜 리스트 & 출결 상태

결제 시 수업 날짜가 4주 단위로 자동 생성되고, 각 날짜에 상태값이 붙는 구조.

### 날짜 리스트 예시

| 회차 | 날짜 | 상태 | 비고 |
|---|---|---|---|
| 1회 | 1/6 (월) | 출석 | |
| 2회 | 1/13 (월) | 결강→보강 | 1/15 화 무성쌤 3시 |
| 3회 | 1/20 (월) | 패스 | 개근 리셋 |
| 4회 | 1/27 (월) | 예정 | |

### 출결 상태 종류
- **출석**: 패드 등원 → 이코딩 횟수 자동 차감
- **지각**: 정각 이후 도착
- **결강**: 불참 + 보강 신청 의사 있음 → 보강 날짜 연결
- **결석**: 불참 + 연락 두절 or 차감 요청 → 수동 차감 처리
- **패스**: 횟수 차감 없이 1회 스킵 → 개근 카운트 리셋
- **긴급패스**: 일반 패스와 별도 유형, 상담원 수동 적용
- **보강완료**: 다른 날짜에 보강 수업 완료
- **마이너스수업**: 결제 없이 차감 선진행, 결제 완료 시 기록 해소

### 출결 오류 케이스 (자동 감지 필요)
- 출석→하원→출석 재등록으로 2회 차감된 경우
- 2개 클래스 수강 중 1개 클래스만 선택하여 출석한 경우
- 이전 결제건이 출결패드에 남아 같이 찍히는 경우
- 수강 종료일이 지나 클래스가 차감 안 되는 경우

## 파생 모듈 2 — 선생님별 시간표

수업 날짜 리스트로부터 선생님별 시간표가 자동 구성된다.

### 시간표 슬롯 규칙
- 한 시간대(슬롯)당 **최대 3명**까지만 배치 가능
- 정규 수강생 + 보강 수강생 **합산** 3명 초과 불가
- 3명 풀이면 해당 슬롯 색상 변경 (초록) → 추가 보강 불가
- 18:00~19:00 브레이크타임 → 수업 및 보강 배치 불가

### 시간표에 표시되는 상태값
- 기본 등록: `홍길동`
- 첫 수업: `홍길동(5.20첫)`
- 복귀: `홍길동(5.20복)`
- 해당 주 휴: `홍길동(5.20휴)`
- 차량 보유: `홍길동(★)`
- 보강 (담당 선생님에게): `홍길동(5.20보)`
- 보강 (타 선생님에게): `홍길동(5.20보/범)` — 원래 선생님 초성 포함
- 특이사항 메모: `홍길동(5.20첫)>수강동의서받기`

### 시간표 오른쪽 메모란
- 자리 대기 중인 수강생 (요청 날짜 + 사유)
- 일정 변경 대기 건

## 파생 모듈 3 — 보강/패스 관리

### 보강

정규 수업에 참여하지 못할 때, 다른 날짜/시간에 수업을 대신 받는 것.
횟수 제한 없음.

**보강 가능 기간**
취소한 주차 기준으로, 앞뒤 정규 수업 사이 기간 내에만 가능.
예) 1/13 수업 취소 시 → 1/6 수업 이후 ~ 1/20 수업 이전 사이에만 보강 가능.

**보강 처리 시 확인 조건**
1. 취소 주차 기준 보강 가능 기간 내인지 확인
2. 해당 시간대 현재 인원이 3명 미만인지 자동 체크
3. 담당 선생님 슬롯 우선 배정, 불가 시 타 선생님

**당일 취소 보강 불가**
당일 취소는 보강 불가 → 차감 또는 패스 유도

**믹싱(1차전직) 보강 — 특수 케이스**
- 담당 선생님(미라쿠도) 승인 필요 → 승인 플로우 별도 존재
- 1차전직: 주차 커리큘럼이 고정되어 있어 같은 주차인 다른 반으로만 보강 가능
- 2차전직: 승인만 받으면 주차 무관하게 보강 가능
- 보강도 불가할 경우 차감 처리

### 패스

정규 수업에 참여 못하고, 보강 기간 내 보강도 불가할 때 횟수 차감 없이 1회 쉬어가는 제도.

**발생 기준**
결제분 1개월당 1회 발생. 결제 기간 내에만 사용 가능.
예) 3개월 결제 시 패스 3회 보유, 한 번에 몰아서 사용도 가능.

**패스 불가 케이스**
- 당일 신청 → 불가, 차감 처리
- 믹싱 수업 → 패스 불가, 보강 또는 차감만 가능

**긴급패스**
일반 패스와 별도 유형. 상담원이 수동으로 적용.

**패스 사용 시 연동**
- 개근 카운트 자동 리셋
- 패스 적용 전 수강생에게 개근 리셋 고지 필요

**2주 이상 선패스 요청 처리 기준**
- 잔여 패스 횟수가 충분하면 허용
- 잔여 패스가 1회뿐인데 2주 이상 선패스 요청 시 → 직전 주차에 다시 신청하도록 안내
  (이유: 그 사이 갑작스러운 결강 발생 시 패스 부족 가능성)

## 휴원·퇴원·복귀 운영 규칙

### 잔금 미납 시
- 예약금 결제 후 다음 주 정규수업 때 잔금 결제 안 될 경우:
  - 마이너스 수업 가능 조건: 6개월 이상 장기수강생 / 급여 지연 / 결제 가능 수단 없음
  - 위 조건에 해당하지 않으면 휴원 처리 → 완납 시 복귀 처리
  - **단, 조건 판단은 상담원이 직접 하며 시스템은 강제 분기 없이 표시만 함**

### 퇴원 처리
- 수강생이 퇴원 요청 시 잔여 횟수 차감하지 않고 보존 휴원으로 진행
  (학원에 악감정 없는 상황에서 선의 표시)
- 시스템: enrollment.status → leave, remaining_on_leave 보존

### 연락두절 자동 휴원
- 기존 수강생 연장결제 무단 연락두절 시:
  - 예외상황(급여지연/결제수단없음/6개월이상 장기수강생) 없으면 다음 날 휴원 처리
  - 시스템: contact_due 도래 후 미응답 시 상담원에게 휴원 처리 알림

### 후기 할인 환불 규칙
- 후기 할인 금액으로 결제 + 수업 1회 진행 + 환불 요청인데 후기 미작성 시
  → 정규수업 금액 기준으로 환불 처리 (할인 미적용 금액 기준)
- 시스템: 환불 처리 화면에서 review_url 미입력 여부 자동 체크 후 경고 표시

### 복귀자 규칙
- 완납한 사람에 한해서만 복귀 처리
- 복귀 전까지 보충수업/연습실/음정수업 이용 불가 (잔여 횟수 있어도)
- 복귀하는 주에 복귀일보다 먼저 보충수업 가능 (단, 같은 주 안에서만)
- 커리큘럼 처음부터가 아니라 상태 체크 후 진도 이어서 진행

### 2주 자리대기
- 신규 수강생에게만 적용
- 결제(예약금 포함)가 이루어져야 자리대기 가능
- 시스템: student.status = pending 상태로 enrollment 생성, 2주 이내 결제 미완료 시 알림

### 휴복귀 통계 기입 타이밍
- 환불 또는 결제가 완료된 이후에만 통계 기입 + 휴복귀 처리
- 시스템: payment.fully_paid = true 또는 refunded = true 시점에 통계 자동 반영

## 파생 모듈 4 — 결제자 자동 계산

수업 날짜 리스트에서 잔여 횟수가 언제 0이 되는지 시스템이 자동 계산.

### 결제자 분류
- **수업 후 결제자**: 당일 출석 후 0회 → 수업 종료 후 카운터에서 결제
- **수업 전 결제자**: 예약금 상태 → 당일 수업 입장 전 차액 결제
- **결제 대기자**: 약속 날짜 DB 등록 → 해당일 연락할 자동 등록

### 가격표
- 클린보컬 1개월: 37만원 / 3개월: 95만원 (16만원 할인)
- 언클린보컬 1개월: 37만원 / 3개월: 95만원 (16만원 할인)
- 믹싱 1개월: 28만원 / 4개월: 99만원 (13만원 할인)
- 작곡 1개월: 35만원 / 3개월: 95만원 (10만원 할인)
- 기타 1개월: 30만원 / 3개월: 77만원 (13만원 할인)
- 예약금: 기본 5만원 (금전 문제 시 그 이하도 가능, 단 상담원 먼저 제시 금지)

### 할인 체계 (결제 시 자동 계산)
- **다개월 할인**: 3개월/4개월 일시 결제 시 전 클래스 할인 적용
- **지인 할인**: 지인 추천 등록 시 추천인/피추천인 양측 각 5만원 할인. 지인 정보를 시스템에 입력하여 연결. 결제 등록 시 추천인과 피추천인 각각의 결제건에 discount가 별도로 생성됨
- **중복 수강 할인**: 2번째 클래스부터 5만원씩 추가 할인 (2클래스=5만원, 3클래스=10만원)
- **후기 할인**: 3개월 수강 후 네이버 카페 후기 작성 시 5만원 할인. 이후 3개월마다 추가 가능. 후기 할인 적용 결제일로부터 7일 이내에 URL을 시스템에 입력해야 함. 7일 내 미입력 시 연락할 리스트에 자동 등록
- **인터뷰 할인**: 6개월 수강 후 유튜브 촬영 완료 시 5만원 할인. 이후 6개월마다 추가 가능. 시스템에서 촬영 완료 O 체크로 적용
- **개근 이벤트**: 패스 없이 12주 개근 시 다음 결제에 수업 1회 무료 추가. 기준 시작일: 2025년 10월 28일

### 결제 처리 후 연동
- 영수증 기록 (결제 건 정보 시스템 내 저장)
- 이코딩 결제건 등록
- 상담 통계 자동 반영
- 환불 처리 시 시스템 내 기록 (카드사 취소, 이체 환불 포함)
- 마이너스 수업 상태인 수강생이 결제 완료 시 마이너스 기록 자동 해소

## 파생 모듈 5 — 연락할 리스트 자동 계산

날짜 기반으로 오늘 연락해야 할 대상이 자동으로 뽑힌다.

### 연락할 대상 유형
- 당일 수업 미출석자 (수업 정각에 자동 감지)
- 결제 대기자 (약속일 도래)
- 휴원자 (복귀 예정일 도래)
- 상품권 만료 1개월 전 수강생
- 후기 할인 작성 기한 도래 수강생 (결제일로부터 7일 내 URL 미입력 시 자동 등록)
- 등록 대기자 (상담 후 미등록, 연락 기한 도래)

### 연락 완료 처리
- DB 연락 여부 자동 업데이트 (O/X)
- 복귀 확정 시: 등록 상태 재원으로 변경, 연락기한 삭제, 결과란에 복귀 수업 일정 기재
- 결제 완료 시: 연락 기한 및 이유 삭제
- 상담 통계 연락할 완료 수 자동 반영

## 파생 모듈 6 — 개근/상품권 트래킹

수업 날짜 리스트 기반으로 자동 계산.

### 개근 이벤트
- 패스 미사용 + 정규 수업 + 보강만으로 12주 연속 출석 달성 시
- 달성 즉시 상담원에게 알림 (선생님에게 직접 가지 않음, 상담원이 전달)
- 다음 결제 시 수업 1회 무료 추가 자동 적용
- 패스 사용 순간 카운트 자동 리셋
- 기준 시작일: 2025년 10월 28일

### 상품권 트래킹
- 24회차 출석 도달 시 자동 감지 → 상담원에게 알림 (선생님에게 직접 가지 않음, 상담원이 전달)
- 지류상품권 직접 제작 후 지급 (시스템 외부 행위, 지급 여부만 시스템에 기록)
- 상품권 사용 기간 6개월, 만료 1개월 전 수강생에게 자동 연락 발송
- 시스템 기록 항목: 발급 여부 / 발급일 / 사용 여부 / 사용 클래스 / 만료일

## 파생 모듈 7 — 통계 및 마감 자동 계산

### 시스템 자동 집계 항목 (결제일 기준 스케줄로 제어 가능한 것들)
- 클래스별 결제 수
- 첫 결제 / 추가 결제
- 결제 개월 수별 분류 (1개월 / 3개월 / 4개월 / 예약금 / 2개월)
- 당일 휴원 수 (시작 전 / 3개월 이하 / 3개월 초과)
- 당일 퇴원 수
- 복귀자 수
- 연락할 완료 수 (휴원자 기준)
- 오또 선생님 재원자 수 (클린 / 비클린 구분 자동 집계)

### 시스템 외부에서 별도 관리 (구글 스프레드시트)
- 신규 인입 수
- 방문 상담 예약 수
- 당일 방문 신규 수
- 방문 전 상담 취소 수

### 마감 톡 자동 생성
- 숫자 항목은 시스템이 자동 채움
- 특이사항 텍스트 (휴퇴원자 상세 사유 등)만 상담원이 직접 입력
- 다음 날 결제 리스트 자동 생성 (잔여 횟수 1 또는 예약금 상태인 수강생)
- 다음 날 보강 리스트 자동 생성

### 수강 종료일 동적 계산
수강 종료일은 고정값이 아니라 Schedule 기반으로 동적으로 계산된다.
- 보강이 잡히면 makeup_date 중 가장 늦은 날짜까지 포함
- 패스로 마지막 수업이 밀리면 그것도 반영
- 즉 ends_at은 저장하는 컬럼이 아니라, schedules에서 가장 마지막 날짜(lesson_date 또는 makeup_date)를 뽑는 계산 메서드로 처리

## 스케줄 변동 1건 발생 시 자동 업데이트 범위

현재는 스케줄 변동 1건에 아래를 전부 수동으로 처리해야 한다:

> 이코딩 / 선생님별 시간표(드라이브) / 수강생 DB(노션) / 포탈 재원 상태 / 상담 통계 시트 / 트레이너방 카톡 보고

시스템에서는 한 번의 처리로 연동 가능한 항목은 자동 반영.
단, **포탈(portal.monemusic.com)은 별도 시스템으로 연동하지 않음** — 포탈 상태는 수동 변경 유지.

## 메인 화면 — 대시보드

새로 계산하는 게 아니라, 시스템 전체에 이미 존재하는 데이터를 한 눈에 모아서 보여주는 화면.
상담원이 아침에 출근해서 대시보드 하나만 켜면 오늘 뭘 해야 하는지, 지금 무슨 상황인지, 놓치면 안 되는 게 뭔지 전부 보이는 구조.

현재 매뉴얼 상 아침 루틴 — 카톡 확인, 시간표 확인, 결제자 리스트 작성, 연락할 확인 — 이 전부 대시보드 하나로 대체됨.

### 실시간 (지금 이 순간)
- 현재 시간대 출석 중인 수강생 목록
- 현재 시간대 주차 대수
- 현재 시간대 미출석자 (아직 안 온 사람) — 연락 필요 여부

### 오늘 일정
- 시간대별 전체 시간표 (선생님별, 정규+보강 포함)
- 오늘 결제 예정자 목록 (수업 전/후 구분)
- 오늘 보강 일정 목록
- 오늘 첫 수업 수강생
- 오늘 복귀 수강생

### 오늘 해야 할 일
- 연락할 대상 목록 (결제대기자 / 휴원자 / 등록대기자 / 상품권 만료 임박 / 후기 작성 기한 도래)
- 마이너스 수업 중인 수강생 (결제 미완료 상태)
- 수강 동의서 미수령 수강생
- 2차 전직서 미수령 수강생

### 오늘 마감용
- 당일 결제 완료 건수 및 금액
- 당일 휴퇴원 수
- 당일 복귀 수
- 개근 달성자

모든 항목은 시스템이 자동 계산하여 표시. 상담원은 별도로 집계하거나 확인할 필요 없음.

## 시스템 범위 외 항목 (명확히 제외)

- 신규 인입 수 / 방문 상담 예약 / 당일 방문 / 취소 → 구글 스프레드시트 별도 관리
- 디디쌤(온라인 강의) 후기 연장 / 환불 → 외부 플랫폼
- 현금 시재 실물 확인 → 수동 입력값으로만 수신
- 카드기 승인/취소 결산용지 → 카드기 별도 하드웨어, 금액 수동 입력
- 영수증 실물 노트 부착 → 시스템 외부 행위, 결제 건 기록만 시스템 안
- 국세청 ARS 현금영수증 취소 → 외부 전화 처리, 해당 건 기록만 시스템 안
- 카드사 직접 전화 취소 → 외부 처리, 환불 기록만 시스템 안
- 수강 후기 플러스톡 업로드 (수, 토) → 외부 작업
- 원격학원 채널 답변 → 외부 채널
- 직원 출퇴근 / 연차 / 급여 관리 → 별도 관리
- 플러스톡 카톡 이름 변경 → 카카오 외부 서비스, 시스템 제어 불가
- 포탈 재원 상태 연동 → 포탈은 독립 시스템으로 유지
